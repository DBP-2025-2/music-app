## 🧪 테스트 방법

### 1. 백엔드 서버 시작

```bash
cd backend
npm run dev
```

✅ 서버 실행: http://localhost:4000

### 2. 프론트엔드 개발 서버 시작

```bash
cd frontend
npm run dev
```

✅ 앱 접속: http://localhost:5173

### 3. 테스트 순서

#### 단계 1: 회원가입

1. 프론트엔드 접속: http://localhost:5173
2. "회원가입" 링크 클릭
3. 이메일, 비밀번호, 닉네임 입력
4. "가입하기" 버튼 클릭
5. ✅ "회원가입 성공!" 메시지 표시
6. ✅ 자동으로 로그인 페이지로 이동

#### 단계 2: 로그인

1. 방금 등록한 이메일과 비밀번호 입력
2. "로그인" 버튼 클릭
3. ✅ "로그인 성공!" 메시지 표시
4. ✅ 자동으로 메인 페이지(Artists)로 이동

#### 단계 3: 인증된 기능 사용

1. ✅ 네비게이션이 보이고 모든 페이지 접근 가능
2. ✅ 오른쪽 상단에 "🚪 로그아웃" 버튼 표시

#### 단계 4: 음악 관리 기능 테스트

1. Artists 페이지: 아티스트 추가/수정/삭제
2. Songs 페이지: 곡 추가/수정/삭제
3. Albums 페이지: 앨범 추가/수정/삭제
4. Playlists 페이지: 플레이리스트 생성 및 곡 추가

#### 단계 5: 로그아웃

1. "🚪 로그아웃" 버튼 클릭
2. ✅ 로그인 페이지로 이동
3. ✅ localStorage 토큰 삭제됨

## 🔍 문제 해결

### 로그인 페이지가 보이지 않음

- 프론트엔드 서버가 실행되고 있는지 확인
- `npm run dev` 실행
- http://localhost:5173 접속

### "이미 사용 중인 이메일" 에러

- 다른 이메일 주소로 시도
- 또는 DB에서 users 테이블 초기화

### API 연결 실패

- 백엔드 서버가 실행 중인지 확인
- `.env` 파일의 `VITE_API_BASE` 값 확인
- `http://localhost:4000` 으로 설정되어야 함

### 로그인 후 페이지가 안 보임

- 브라우저 콘솔에서 에러 확인
- localStorage에서 authToken 확인
- 네트워크 탭에서 API 응답 확인
